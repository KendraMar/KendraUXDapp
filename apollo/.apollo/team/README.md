# Team Folder

This folder contains team-level context that applies across all features and releases.

## Structure

```
team/
├── README.md           # This file
├── context-sources.md  # Team resources (calendars, channels, etc.)
├── guidelines/         # Design guidelines and standards
├── research/           # UX research artifacts
├── personas/           # User personas
└── templates/          # Team-specific templates
```

## Guidelines

The `guidelines/` folder should contain design guidelines and standards that the team follows. This might include:
- Visual design guidelines
- Interaction patterns
- Accessibility standards
- Content/writing guidelines

## Research

The `research/` folder should contain:
- User research findings
- Usability test results
- Survey data
- Competitive analysis

## Personas

The `personas/` folder should contain user persona documentation:
- Primary personas
- Secondary personas
- Anti-personas (who we're NOT designing for)

## Context Sources

The `context-sources.md` file should document team-wide resources:
- Team calendars
- Slack channels
- Shared drives
- Jira projects
