# AI Summary Feature Implementation

## Overview
This document describes the implementation of the AI-powered summarization feature for Feed and Tasks pages. The feature integrates with a local OpenAI-compatible API to generate brief summaries of messages and issues.

## Configuration

### config.json
Added AI configuration section:
```json
{
  "ai": {
    "apiUrl": "http://127.0.0.1:1234",
    "model": "openai/gpt-oss-20b"
  }
}
```

**User-configurable fields:**
- `apiUrl`: The URL of your local OpenAI-compatible API endpoint
- `model`: The model identifier to use for summarization

## File Changes

### 1. Data Files

#### `data/tasks/tasks.json` (NEW)
- Created to store AI summaries for Jira issues
- Structure: `{ "issues": [{ "key": "ISSUE-KEY", "aiSummary": "summary text" }] }`

#### `data/feed/feed.json` (MODIFIED)
- Removed all dummy `aiSummary` fields
- Summaries are now dynamically generated and stored

### 2. Server (server.js)

#### New Functions:
- `loadConfig()`: Loads the entire config file
- `loadAiConfig()`: Returns AI configuration section
- `makeAiRequest(aiConfig, prompt)`: Makes requests to the AI API

#### New API Endpoints:

**POST /api/feed/:id/summarize**
- Generates AI summary for a feed item
- Saves summary to `data/feed/feed.json`
- Returns: `{ success: true, summary: "..." }`

**POST /api/tasks/:key/summarize**
- Generates AI summary for a Jira issue
- Saves summary to `data/tasks/tasks.json`
- Returns: `{ success: true, summary: "..." }`

**GET /api/tasks** (MODIFIED)
- Fetches Jira issues and merges with cached AI summaries
- Returns issues with `aiSummary` field (null if not yet generated)

### 3. Frontend Components

#### Feed.js
**New State:**
- `summarizing`: Boolean indicating if AI request is in progress
- `summaryError`: Error message from AI request

**New Function:**
- `generateSummary(itemId)`: Calls API to generate/regenerate summary

**UI Changes:**
- Shows "Summarize with local AI" button when no summary exists
- Shows summary with "Regenerate" button when summary exists
- Loading state during summarization
- Error display if summarization fails

#### Tasks.js
**New State:**
- `summarizing`: Boolean indicating if AI request is in progress
- `summaryError`: Error message from AI request

**New Function:**
- `generateSummary(issueKey)`: Calls API to generate/regenerate summary

**UI Changes:**
- Shows "Summarize with local AI" button when no summary exists
- Shows summary with "Regenerate" button when summary exists
- Loading state during summarization
- Error display if summarization fails
- Changed data source from `/api/jira/issues` to `/api/tasks` to get cached summaries

## How It Works

### Workflow for Feed Items:
1. User clicks on a feed message
2. If `aiSummary` exists in data, display it with "Regenerate" button
3. If no summary exists, show "Summarize with local AI" button
4. When button clicked:
   - Send POST request to `/api/feed/:id/summarize`
   - Server constructs prompt from message content
   - Server calls local AI API
   - Server saves summary to `data/feed/feed.json`
   - Frontend updates display with new summary

### Workflow for Tasks/Issues:
1. User clicks on a Jira issue
2. If `aiSummary` exists in cached data, display it with "Regenerate" button
3. If no summary exists, show "Summarize with local AI" button
4. When button clicked:
   - Send POST request to `/api/tasks/:key/summarize` with issue details
   - Server constructs prompt from issue fields
   - Server calls local AI API
   - Server saves summary to `data/tasks/tasks.json`
   - Frontend updates display with new summary

## AI Prompt Template

### For Feed Messages:
```
Briefly summarize this message in under 100 words. Focus on key points and action items:

Subject: [subject]
From: [from]
Content: [content]
```

### For Jira Issues:
```
Briefly summarize this Jira issue in under 100 words. Focus on the problem, impact, and what needs to be done:

Issue: [key]
Type: [issueType]
Priority: [priority]
Status: [status]
Summary: [summary]
Description: [description]
```

## Error Handling

- Configuration missing: Returns 400 with error message
- AI API connection failed: Returns 500 with error message
- Item/issue not found: Returns 404 with error message
- Frontend displays errors below the button

## Storage

- Feed summaries: Stored in `data/feed/feed.json` (same file as feed items)
- Task summaries: Stored in `data/tasks/tasks.json` (separate cache file)
- Summaries persist across page reloads
- Summaries can be regenerated by clicking "Regenerate" button

## Benefits

1. **On-demand**: Summaries generated only when needed (saves API costs/time)
2. **Cached**: Once generated, summaries are stored and reused
3. **Local AI**: Uses local model for privacy and speed
4. **User control**: Users decide when to generate/regenerate summaries
5. **Configurable**: Easy to change AI endpoint and model in config.json
