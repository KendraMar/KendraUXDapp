# Static File Serving

**Implementation:** Express static middleware in `server.js`

## Overview

The Express server serves compiled React application files and static assets from the `dist/` directory.

## Configuration

```javascript
app.use(express.static(path.join(__dirname, 'dist')));
```

## Served Files

The `dist/` directory contains:

### Compiled JavaScript
- `bundle.[contenthash].js` - Main application bundle
- `bundle.[contenthash].js.map` - Source maps for debugging

### Assets
- Font files (`.woff2` format)
- `favicon.ico`
- `index.html` - HTML template

## Build Process

Files are generated by Webpack during the build process:
1. JavaScript files are bundled and minified
2. Assets are processed and copied
3. HTML is generated with proper asset references
4. Content hashes are added for cache busting

## Fallback Route

For client-side routing, all non-API routes serve `index.html`:

```javascript
app.get('*', (req, res) => {
  res.sendFile(path.join(__dirname, 'dist', 'index.html'));
});
```

This ensures:
- Direct URL access works (e.g., `/dashboard`)
- Page refreshes don't result in 404 errors
- React Router can handle all client-side routes

## Public Path

- **Public Path**: `/` (root)
- All assets are served from the root path
- Webpack configured with `publicPath: '/'`

## Development vs Production

- **Development**: Webpack watch mode rebuilds on changes
- **Production**: Pre-built files served from `dist/`
