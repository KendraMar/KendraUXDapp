# Agent Context

Guidance for AI assistants working with design files in Apollo.

## Structure

```
agents/
├── rules/             # Behavioral guidelines
│   └── design-guidelines.md
└── skills/            # Specialized capabilities
    ├── review-design/
    ├── check-design-system-compliance/
    └── event-tracking/
```

## Rules

**design-guidelines.md** — How to write design history entries, follow PatternFly design system standards, and maintain accessibility in Apollo.

## Skills

**review-design** — Comprehensive design critique covering usability, visual design, accessibility, and PatternFly compliance.

**check-design-system-compliance** — Verify a page or component follows PatternFly 6 design system standards.

**event-tracking** — Track design events and update the appropriate design-history.md files.
