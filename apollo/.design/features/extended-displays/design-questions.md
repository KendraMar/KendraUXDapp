# Extended Displays — Design Questions

## Open

### Should display coordination be opt-in or automatic?
- **Raised**: 2026-02-10
- **Context**: When multiple Apollo windows are detected, should they automatically begin coordinating, or should the user explicitly "link" them? Automatic is more magical but could be surprising. Opt-in is safer but adds friction.

### What is the minimum viable experience?
- **Raised**: 2026-02-10
- **Context**: The brainstorm covers many concepts (coordinated navigation, distributed interface, context radiators, workstation presets, AI-augmented display management). Which single use case delivers the most value with the least complexity?

### How should Extended Displays interact with Spaces?
- **Raised**: 2026-02-10
- **Context**: If a user switches Spaces on one window, should all linked windows switch too? Should each window be independently assignable to a different Space? What's the default behavior?

### What's the right UI for display management?
- **Raised**: 2026-02-10
- **Context**: Should there be a visible display manager panel, a masthead icon, a keyboard shortcut menu, or should it be mostly invisible? How do we surface multi-display capabilities without cluttering the single-display experience?

### How do we handle the privacy implications of peripheral displays?
- **Raised**: 2026-02-10
- **Context**: In open offices or shared spaces, secondary displays may show sensitive information. Should there be a privacy/blur mode? How do we balance ambient awareness with information security?
